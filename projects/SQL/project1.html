SweepShift: Refining Layoffs Data for Analysis

Data Cleaning Data Analysis

Project Overview
The objective of this project is to perform data cleaning on a dataset named "layoffs" using SQL. The dataset contains information about layoffs, including details such as company name, location, industry, total number of employees laid off, percentage of employees laid off, date of layoffs, stage of the company, country, and funds raised in millions. The data cleaning process involves several steps such as removing duplicates, standardizing data, handling null or blank values, and performing exploratory data analysis.

Project Details

Removing Duplicates:
  • Identifying and removing duplicate records based on specific columns such as company, location, industry, total laid off, percentage laid off, date, stage, country, and funds raised.
Standardizing Data:
  • Standardizing company names by removing leading and trailing spaces.
  • Standardizing industry names by categorizing similar industries (e.g., grouping all "Crypto" related industries under a single category).
  • Standardizing country names by removing trailing punctuation marks and abbreviations.
Working on Time Series Data:
  • Converting the date column from string to date format using appropriate functions.
  • Altering the table to modify the date column datatype to DATE.
Handling Null and Blank Values:
  • Identifying and handling null and blank values in columns such as total laid off, percentage laid off, and industry.
  • Attempting to fill null values using joins with existing data where applicable.
Exploratory Data Analysis (EDA):
  • Performing EDA to gain insights into the dataset.
  • Analyzing total layoffs by company, industry, funding stage, and year.
  • Calculating rolling totals of layoffs over time.
  • Ranking companies based on layoffs per year and selecting the top 5 companies for each year.
