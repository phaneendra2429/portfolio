Image Gallery Application using Flask in GCP

GCP Flask Firebase

The Image Gallery Application is a web-based platform designed to allow users to securely upload, manage, and view images. It offers user authentication, image storage utilizing Google Cloud Storage, and deployment through Google Cloud Run. The application is built using Python Flask for the backend, incorporating Firebase Authentication for user authentication and Google Cloud services for image storage and deployment.

Project Details

Tools and Technologies Used:
  • Flask: Flask is a lightweight WSGI web application framework in Python. It provides tools and libraries for building web applications.
  • Firebase Authentication: Firebase Authentication is a service provided by Google for authenticating users to your app. It offers multiple authentication methods, including email/password, Google sign-in, and more.
  • Google Cloud Platform (GCP):
    • Google Cloud Storage: A scalable object storage service for storing and retrieving data. It provides durability, scalability, and security for storing user-uploaded images.
    • Google Cloud Datastore: A NoSQL document database built for automatic scaling, high performance, and ease of application development.
    • Google Cloud Run: A managed compute platform that enables users to run stateless containers on a fully managed environment.
Detailed Functionality:
  1. User Authentication:
    • The application utilizes Firebase Authentication for user authentication.
    • Users can sign up using their email and password, and existing users can log in securely.
  2. Image Upload and Storage:
    • Users can upload images through the web interface.
    • Upon upload, images are securely stored in Google Cloud Storage within individual user-specific folders.
    • Metadata about the uploaded images, including user information and image URL, is stored in Google Cloud Datastore.
  3. Image Viewing and Management:
    • Users can view their uploaded images in the gallery interface.
    • Images are displayed along with their metadata, including EXIF data if available.
    • Users can delete images from their gallery.
Deployment:

The application is deployed on Google Cloud Platform using Cloud Run:

  • Docker containers are created for the Flask application.
  • Implemented CI/CD by integrating github and google cloud Run
  • The containerized application is deployed to Google Cloud Run, which automatically manages scalability and availability.
